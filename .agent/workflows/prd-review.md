# /prd-review - PRD Review & Optimization

$ARGUMENTS

---

## Task

Analyze the PRD, identify gaps, and align on goals before starting execution.

---

## Steps

1. **Load PRD**
   - Prefer `PRD.md` or any `*PRD*.md` in root or `Docs/`

2. **Summarize Intent**
   - Core goals, audience, scope, success metrics

3. **Gap Analysis**
   - Missing requirements, unclear scope, risks

4. **Optimization Suggestions**
   - Clarify scope, MVP, priorities
   - Improve measurability

5. **Questions**
   - Ask only the critical questions

---

## Output

- PRD summary
- Gaps & risks
- Optimization suggestions
- Clarifying questions

---

## Usage

```
/prd-review
```
